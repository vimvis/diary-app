# Diary App Next Step

## 목표

현재 단일 일기 작성 앱을 다음 단계의 "회고 중심 개인 다이어리"로 확장한다.

핵심 방향:

- 로컬 저장 안정성 강화
- 기록 입력 유연화
- 회고 탐색 경험 강화
- 감정 데이터 기반 검색/요약 추가

## 개발 우선순위

### 1단계: 핵심 사용성 개선

1. 가변 일기 항목 수
2. 로컬 저장 구조 개선
3. 검색 기능
4. 랜덤 회고 기능

### 2단계: 회고 경험 강화

5. 감정 선택 및 강도 입력
6. 기간별 요약
7. 달력/리스트 UI 개선

### 3단계: 데이터 안정성과 확장

8. 백업/복원
9. 통계/패턴 시각화
10. 자동저장/임시복구

## 정보 구조

### 화면 구성

1. 오늘 작성
2. 기록 탐색
3. 달력
4. 요약
5. 설정

### 각 화면 역할

#### 오늘 작성

- 오늘 날짜 기준 기록 작성
- 1개 이상, 원하는 만큼 항목 추가 가능
- 감정 선택
- 자동저장/수동저장 상태 표시

#### 기록 탐색

- 키워드 검색
- 감정 필터
- 날짜 범위 필터
- 랜덤 기록 보기

#### 달력

- 기록 유무 표시
- 감정 색상 또는 밀도 표시
- 날짜 클릭 시 상세 보기

#### 요약

- 최근 7일 요약
- 최근 30일 요약
- 사용자 지정 기간 요약

#### 설정

- 데이터 내보내기/가져오기
- 테마 설정
- 로컬 데이터 초기화

## 데이터 모델 제안

```ts
type DiaryEntry = {
  id: string;
  date: string; // YYYY-MM-DD
  items: string[];
  emotions: {
    value: string;
    intensity: number; // 1-5
  }[];
  tags: string[];
  createdAt: string;
  updatedAt: string;
};
```

추가 저장소:

```ts
type DiaryStore = {
  entries: Record<string, DiaryEntry>;
  settings: {
    theme: "light" | "dark";
    autoSave: boolean;
  };
};
```

## 기능 명세

### 1. 가변 항목 수

목표:

- 사용자가 1개만 작성하거나 여러 개를 추가 작성할 수 있어야 한다.

세부 요구사항:

- 기본 입력칸은 3개로 시작
- "항목 추가" 버튼 제공
- 항목이 1개 이상일 때만 삭제 가능
- 빈 항목은 저장 시 자동 제거

UI 제안:

- 각 항목 우측에 삭제 버튼
- 마지막 항목 아래에 `+ 기억 추가`

### 2. 로컬 저장 구조 개선

목표:

- 데이터 증가에도 안정적으로 저장/조회 가능해야 한다.

세부 요구사항:

- 초기 버전은 `localStorage` 유지 가능
- 다음 단계에서 `IndexedDB` 전환
- 저장 실패 시 토스트 표시
- 마지막 저장 시각 표시

### 3. 검색 기능

목표:

- 사용자가 과거 기록을 빠르게 찾을 수 있어야 한다.

검색 조건:

- 본문 키워드
- 감정 태그
- 날짜 범위
- 태그

검색 결과 UI:

- 날짜
- 감정
- 일치 문장 일부 미리보기
- 클릭 시 상세 열기

### 4. 랜덤 회고

목표:

- 기록 탐색을 의도적 회고 경험으로 바꾼다.

기능 형태:

- 전체 기록 중 랜덤
- 이번 달 중 랜덤
- 1년 전 오늘
- 특정 감정의 기록 중 랜덤

UI 제안:

- 탐색 화면 상단에 `랜덤 기록 보기`
- 결과를 카드 또는 모달로 표시

### 5. 감정 표현

목표:

- 기록에 감정을 구조적으로 남길 수 있어야 한다.

세부 요구사항:

- 감정 칩 선택
- 복수 선택 가능
- 감정 강도 1~5 선택

기본 감정 예시:

- 기쁨
- 감사
- 평온
- 불안
- 피곤
- 슬픔
- 성취감
- 외로움

### 6. 특정 기간 요약

목표:

- 일정 기간의 기록을 빠르게 회고할 수 있어야 한다.

초기 구현:

- 총 기록 수
- 가장 많이 등장한 감정
- 자주 나온 단어
- 자주 언급한 주제
- 대표 문장 3개

추천 기간:

- 최근 7일
- 최근 30일
- 이번 달
- 사용자 지정

초기 요약 방식:

- 규칙 기반 빈도 분석
- 불용어 제거
- 감정 집계

## 바로 개발할 MVP 범위

지금 바로 구현할 때 가장 효율적인 범위:

1. 항목 가변화
2. 감정 선택
3. 검색
4. 랜덤 기록 보기
5. 최근 7일/30일 요약

## 구현 시 주의점

- 기존 `item1`, `item2`, `item3` 구조를 배열 구조로 마이그레이션해야 함
- 검색과 요약을 고려하면 저장 시 정규화가 필요함
- 감정/태그 UI는 모바일에서 과밀해지지 않게 접기 구조가 필요함
- 요약은 처음부터 AI 의존으로 만들지 말고 로컬 규칙 기반으로 시작하는 편이 안정적임
